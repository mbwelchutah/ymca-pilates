Fix two issues in the Playwright bot:

1. Class matching is too strict and fails even when the class is visible
2. Status messaging incorrectly reports “not found” when the class is actually present but booking is not open

---

## GOAL

Make the bot:

* reliably find the visible class row on the schedule
* correctly report:

  * “found but not open yet”
  * instead of “not found”

---

## DO NOT

* change scheduler timing
* change booking window logic
* change DB schema
* remove verification logic

---

# PART 1 — Fix class matching (critical)

The class row is already visible in the UI:

* Core Pilates
* 7:45 a - 8:45 a
* Stephanie S.

So matching must be **fuzzy + user-visible based**, NOT exact.

### Replace existing strict matching with scoring logic:

1. Collect visible rows/cards for the selected day

2. For each row:

   * get `innerText().toLowerCase()`

3. Score each row:

* title contains "core pilates" → +5
* time contains "7:45" → +5
* instructor contains "stephanie" → +3

4. Normalize:

* treat "7:45 AM" and "7:45 a" as equivalent
* match instructor by first name only

5. Log all rows:

```js
console.log("Row:", text.slice(0, 120), "Score:", score);
```

6. Select highest score

7. If best score < threshold → THEN it's truly "not found"
