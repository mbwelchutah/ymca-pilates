Add “tap thumbnail → replay last run (mini trace)” so clicking a failure screenshot shows contextual details of that run.

Current state:

* failure panel shows last 5 screenshots
* filenames include reason + timestamp
* screenshots stored locally

Goal:
When clicking a failure thumbnail, show a lightweight “trace replay”:

* screenshot (large)
* reason (time / instructor)
* expected vs actual values
* modal text preview (if available)

Do NOT:

* implement full Playwright tracing
* add database
* over-engineer

---

# PART 1 — Save metadata alongside screenshot

When saving screenshot, ALSO save JSON:

```js
const meta = {
  jobId: job.id,
  reason: reasonTag,
  expectedTime,
  expectedInstructor,
  modalPreview: modalText.slice(0, 300),
  timestamp: new Date().toISOString()
};

const metaFile = filename.replace(".png", ".json");

fs.writeFileSync(metaFile, JSON.stringify(meta, null, 2));
```

# PART 2 — Extend /api/failures

Return metadata too:

```js
const metaPath = path.join(dir, name.replace(".png", ".json"));

let meta = {};
if (fs.existsSync(metaPath)) {
  meta = JSON.parse(fs.readFileSync(metaPath));
}

return {
  name,
  reason,
  time,
  meta
};
```

# PART 6 — Render trace details

```js
function openTrace(f) {
  const modal = document.getElementById("trace-viewer");
  const img = document.getElementById("trace-image");
  const details = document.getElementById("trace-details");

  img.src = `/screenshots/${f.name}`;

  details.innerHTML = `
    <div><strong>Reason:</strong> ${f.meta.reason}</div>
    <div><strong>Expected Time:</strong> ${f.meta.expectedTime}</div>
    <div><strong>Instructor:</strong> ${f.meta.expectedInstructor}</div>
    <div><strong>Preview:</strong><br/>${f.meta.modalPreview}</div>
  `;

  modal.classList.remove("hidden");
}
```

---

# PART 7 — Close modal

```js
document.getElementById("trace-viewer").onclick = () => {

};
```
