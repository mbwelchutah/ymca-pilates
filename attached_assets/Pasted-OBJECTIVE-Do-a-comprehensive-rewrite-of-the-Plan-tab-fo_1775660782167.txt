OBJECTIVE

Do a comprehensive rewrite of the Plan tab for the YMCA Booker app.

The Plan tab should no longer feel like a passive list of saved jobs.
It should become a clean, mobile-first planning pipeline that works seamlessly with the Now screen.

CORE PRODUCT DIRECTION

Now = execution / live state / current target  
Plan = upcoming classes / scheduling intent / what happens next

Plan should answer:
- what classes are coming up?
- when will each one open?
- which one is next?
- what happens if I tap it?
- how do I edit or remove it quickly?

The Plan tab must feel:
- simple
- obvious
- iPhone-friendly
- tightly connected to Now
- calm and modern
- Apple-clean

Do NOT make it feel like a database or admin panel.

--------------------------------------------------
PRIMARY DESIGN RULES
--------------------------------------------------

1. Plan and Now must feel like one workflow
   - selecting a class in Plan should move to Now
   - Now should show that selected class as the active focus

2. Default sort must be:
   - by date/time
   - earliest upcoming class first

3. Optional sort choices:
   - Date
   - Class name
   - Instructor

4. Each card must show:
   - class name
   - day/date
   - class time
   - instructor
   - open time / opens in
   - status
   - visible edit
   - visible delete

5. Edit and delete must be more prominent than they are now

6. “Add” is currently in the wrong place
   - remove it from the top-right header
   - replace with a better, context-aware add flow inside the Plan screen

7. Browse / refresh errors or technical messages should not float awkwardly in Plan
   - if needed, use clean inline empty-state wording
   - do not show technical error clutter in the main Plan list

8. After successful registration, the recurring class should automatically advance one week
   - the completed occurrence should no longer be the active future target
   - the next weekly occurrence should become the planned one

--------------------------------------------------
TARGET UX MODEL
--------------------------------------------------

Plan should feel like:

“Here is my upcoming booking queue.”

NOT:

“Here is a list of records.”

Top of Plan:
- title / lightweight summary
- next upcoming booking timing
- Add Class CTA
- sort control

Main body:
- upcoming class cards
- earliest first

Tap card behavior:
- tapping a class card should make it the active target
- app should navigate to Now
- Now should reflect that selected class immediately
