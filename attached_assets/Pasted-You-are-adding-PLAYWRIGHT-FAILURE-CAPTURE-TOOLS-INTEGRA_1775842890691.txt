You are adding PLAYWRIGHT FAILURE CAPTURE + TOOLS INTEGRATION to the YMCA Booker app.

PRIMARY GOAL

When a Playwright-based run fails or becomes uncertain, automatically capture a screenshot and make it available from the Tools tab.

This should make debugging dramatically easier without cluttering the main UI.

The user should be able to:
- see that a run failed
- open Tools
- see the failed run
- tap to view the screenshot

IMPORTANT RULES

- Keep app runnable after every stage
- Do not break booking/preflight logic
- Do not capture screenshots on every successful step by default
- Do not flood storage
- Do not show raw technical logs in the main UI by default
- Keep screenshot access clean and secondary
- Stop after each stage and summarize exactly what changed
- Do not jump ahead
