Add a “click by index fallback” to the Playwright bot so it can still select the correct class when text matching fails.

Current state:

* primary match uses time + fuzzy instructor
* sometimes fails due to UI differences (abbreviated names, missing titles)

Goal:
If no match is found, fall back to clicking the correct row by index.

Do NOT:

* remove existing matching logic
* hardcode a single index
* break existing success path

---

## PART 1 — Collect all visible class rows

```js
const rows = await page.locator('div:has-text(" a - ")').all();
```

(or your existing selector for class rows)

---

## PART 2 — Try primary match first

```js
let matchedRow = null;

for (const row of rows) {
  const text = (await row.innerText()).toLowerCase();

  if (text.includes(normalizedTime) && matchesInstructor(text, instructor)) {
    matchedRow = row;
    break;
  }
}
```

---

## PART 3 — Add index fallback

If no match found:

```js
if (!matchedRow && rows.length > 0) {
  console.log("⚠️ No exact match — using index fallback");

  // safest fallback: first row (or configurable)
  matchedRow = rows[0];
}
```
