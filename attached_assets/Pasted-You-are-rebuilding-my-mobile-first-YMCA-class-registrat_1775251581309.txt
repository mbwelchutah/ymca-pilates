You are rebuilding my mobile-first YMCA class registration app in React + Tailwind.

This is a product-guided rebuild, not just a visual refresh.

Use:
- React
- TypeScript
- Tailwind CSS

Primary target:
- iPhone-first web app
- clean, modern, calm interface
- subtle Strava-like fitness app discipline
- but primarily a focused gym class registration utility

CORE PRODUCT PURPOSE

This app exists to:
register for a gym class precisely when registration opens.

It is not a generic bot dashboard.
It is not an automation toy.
It is not a developer control panel.

Every screen, label, and component should reinforce this product purpose:
- watch a class
- wait for registration to open
- register at the exact right time
- let the user trust the process

DESIGN DIRECTION

The app should feel:
- clean
- modern
- iPhone-oriented
- calm
- precise
- trustworthy
- fitness-adjacent
- quietly premium

It should NOT feel:
- retro
- gimmicky
- overloaded with icons
- dev-oriented
- like a monitoring dashboard
- like a page full of controls

VISUAL RULES

Do not use:
- rockets
- oversized gears
- “Live Mode Active”
- bright red primary actions
- raw backend labels
- repeated status chips
- repeated duplicate cards
- heavy dashboard segmentation

Use:
- soft white / warm neutral backgrounds
- dark text
- restrained accent color
- rounded iPhone-style cards
- strong typography hierarchy
- calm spacing
- minimal but useful iconography
- clean touch-friendly controls

APP STRUCTURE

Use exactly 3 primary tabs:
1. Now
2. Plan
3. Settings

Do not use “Tools” as a primary navigation label.

Do not collapse all content into one screen.

Each tab must keep a clear responsibility.

TAB RESPONSIBILITIES

NOW
Purpose:
Show the active class currently being watched and the exact booking readiness state.

PLAN
Purpose:
Show active and future class bookings, and allow add/edit/delete.

SETTINGS
Purpose:
Hold detailed development controls, app behavior, dry run, debugging, and future scraping-related settings.

Do not mix those responsibilities.

TECHNICAL DIRECTION

Build this as a React app with:
- reusable components
- clear screen boundaries
- typed props
- clean state handling
- mobile-first layout
- Tailwind utility styling
- a lightweight design system

If needed, create a src/features structure or similar organization.

Recommended architecture:
- src/components
- src/screens
- src/features
- src/types
- src/lib
- src/data or src/state

The rebuild should preserve the underlying business logic as much as possible, but the UI architecture should become much cleaner.
