You are polishing the Now tab of the YMCA Booker app so it works seamlessly with the redesigned Plan tab.

PRIMARY GOAL

Make the Now tab feel like:
- the live execution view
- the current active target
- calm, trustworthy, and obvious
- tightly connected to Plan

Plan now acts like the upcoming booking queue.
Now must clearly reflect the currently selected class from Plan.

The user should feel:
- “This is the class the system is focused on now.”
- “I can see what happens next.”
- “I do not need to decode the interface.”

IMPORTANT PRODUCT DIRECTION

Plan = upcoming queue
Now = active selected item from Plan

Now should answer only:
1. What class is active right now?
2. When does booking open?
3. What state is the system in?
4. Is the system ready?
5. Do I need to do anything?

Do NOT let Now become a debug panel.
Do NOT surface technical clutter in the main view.
Do NOT show overlapping signals that say the same thing.

STRICT RULES

- Keep app runnable after every stage
- Do not break current booking logic
- Do not break current Plan → Now card selection flow
- Preserve strong trust/reliability signals, but simplify presentation
- Stop after each stage and summarize exactly what changed
- Do not jump ahead
