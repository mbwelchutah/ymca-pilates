You are redesigning the Tools tab of the YMCA Booker app.

PRIMARY GOAL

Simplify the Tools tab so it feels clean, useful, and calm.

Right now Tools contains too much at once:
- run history
- automation toggles
- suggestions
- failure trends
- recent failures
- technical diagnostics

This makes it feel crowded and harder to understand.

The new Tools tab should answer only these 3 questions first:

1. What happened last time?
2. Are the automations healthy?
3. What needs my attention?

Everything else should be secondary or collapsed.

IMPORTANT PRODUCT DIRECTION

Now screen = trust + current booking state
Tools = explanation + health + diagnostics

Tools should feel like:
- an operations summary
- a health dashboard
- a clean diagnostic center

NOT:
- a raw error dump
- a giant analytics page
- a debug wall

STRICT RULES

- Keep app runnable after every stage
- Do not change booking logic
- Do not remove data entirely unless explicitly stated
- Prefer collapsing / demoting over deleting
- Keep mobile-first layout
- Stop after each stage and summarize exactly what changed
- Do not jump ahead

==================================================
NEW TOOLS TAB STRUCTURE
==================================================

Target order, top to bottom:

1. Last Run
2. Automation Health
3. Needs Attention
4. Recent Activity (collapsed or compact)
5. Technical Details (collapsed)
6. Failure Trends (collapsed, optional)

Tools should not open with a wall of failures.
