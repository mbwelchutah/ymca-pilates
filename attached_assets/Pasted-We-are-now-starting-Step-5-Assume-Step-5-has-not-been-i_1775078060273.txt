## Step 5 requirements

Build a **simple jobs dashboard** in the existing web app.

### Main goals

1. Show **all saved jobs** from the database on the page
2. Make it possible to **choose and run a specific job**
3. Improve the visual layout so the page feels like a real control panel
4. Keep the patch beginner-friendly and do not over-engineer

## What to build

### A. Jobs list / dashboard

On the main page, render all saved jobs from SQLite.

For each job, show:

* id
* class title
* day of week
* class time
* instructor
* active / inactive status

### B. Selected job area

At the top of the page, show a clear “selected job” summary card.

When a user clicks a job in the list, update the selected-job summary.

### C. Run button for selected job

Add a button that runs the currently selected job.

This should call a route like:

* `/run-job?id=<selectedJobId>`

### D. Interface improvement

Improve the layout so it is easier to understand:

* a clean card-style layout
* clearer typography
* spacing between sections
* a visible selected-job panel
* better separation between actions and status messages
* mobile-friendly width and spacing

Keep it simple:

* plain server-rendered HTML/CSS is fine
* no framework rewrite
* no React
* no major architecture changes

### E. Status/result area

Keep a single status box that shows:

* running
* success
* error

This should update when a job is run.
