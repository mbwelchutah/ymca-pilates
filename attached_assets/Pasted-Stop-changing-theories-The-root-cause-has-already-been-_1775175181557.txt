## Facts already established

1. The class row is already visible on the schedule before booking opens.
   Example on Wed 08:

   * Core Pilates
   * 7:45 a - 8:45 a
   * Stephanie S.

2. The Playwright browser was rendering times in UTC, causing a +7 hour shift:

   * 7:45 a PDT appeared as 2:45 pm in the bot modal
   * 10:35 a appeared as 5:35 pm
   * 11:45 a appeared as 6:45 pm

3. The correct fix is to set the Playwright browser context timezone to:

   * `America/Los_Angeles`

4. There was also a confirmed time-regex bug:

   * `\b0?7:45\s*a` failed on combined card text like `Eugene Y7:45 a`
   * the correct match is the looser pattern using the visible time text, e.g. `/7:45\s*a/i`

5. After these fixes, the correct pre-open result is:

   * `found_not_open_yet`
   * because the class is found, but registration is not open yet
