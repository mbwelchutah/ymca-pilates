Make dropdown selection fully reliable by handling animation, visibility, and dynamic loading correctly.

Current issue:

* clicking hidden <option>
* dropdown not ready when clicking
* intermittent timeout errors

Goal:
Make dropdown selection deterministic and stable.

---

# PART 1 — ALWAYS target visible trigger (not option)

```js
const dropdown = page.getByText("Category", { exact: true });
```

---

# PART 2 — Wait until trigger is ready

```js
await dropdown.waitFor({ state: "visible" });
await dropdown.scrollIntoViewIfNeeded();
```

👉 avoids clicking too early ([browserstack.com][1])

---

# PART 3 — Click trigger

```js
await dropdown.click();
```

---

# PART 4 — WAIT for dropdown to OPEN (CRITICAL)

```js
const option = page.getByText("Yoga/Pilates", { exact: false });

await option.waitFor({ state: "visible" });
```

👉 do NOT skip this — most failures happen here

---

# PART 5 — Ensure option is interactable

```js
await option.scrollIntoViewIfNeeded();
await expect(option).toBeVisible();
```

---

# PART 6 — Click option

```js
await option.click();
```

---

# PART 7 — VERIFY selection worked (IMPORTANT)

```js
await expect(dropdown).toContainText("Yoga");
```

---

# PART 8 — Add retry wrapper (production-safe)

```js
async function selectDropdown(label, value) {
  const dropdown = page.getByText(label);

  await dropdown.waitFor({ state: "visible" });
  await dropdown.click();

  const option = page.getByText(value);

  await option.waitFor({ state: "visible", timeout: 3000 });

  await option.click();
}
```

---

# PART 9 — Optional: handle slow animation

```js
await page.waitForTimeout(150);
```

(use only if needed — avoid long waits ([Autify][2]))

---

# PART 10 — Debug fallback

If still flaky:

```js
console.log("Dropdown open state:");
console.log(await page.content());
```

---

# FINAL FLOW

1. wait for dropdown visible
2. click dropdown
3. wait for options visible
4. click option
5. verify selection
