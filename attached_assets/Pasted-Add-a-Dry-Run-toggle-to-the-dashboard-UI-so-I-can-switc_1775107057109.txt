Add a “Dry Run” toggle to the dashboard UI so I can switch between test mode and real booking without changing code.

Current state:

* bot supports DRY_RUN via environment variable or flag
* UI has buttons but no mode toggle
* app will be used primarily on iPhone

Goal:
Provide a simple, iOS-style toggle to control dry run mode globally.

Do NOT:

* change booking logic structure
* add frameworks
* over-engineer

---

## PART 1 — Add global dry run state (server)

In server.js:

1. Add:

```js
let dryRunEnabled = true;
```

2. Export setter:

```js
module.exports.setDryRun = (value) => {
  dryRunEnabled = value;
};
```

3. When running jobs (ALL paths):

* pass:

```js
runBookingJob(job, { dryRun: dryRunEnabled });
```

## PART 2 — Add routes

POST `/set-dry-run`

Logic:

* read boolean from request

## PART 5 — Client-side logic

```js
const toggle = document.getElementById("dry-run-toggle");

toggle.addEventListener("change", async () => {
  const enabled = toggle.checked;

  await fetch("/set-dry-run", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ enabled })
  });

  updateDryRunUI(enabled);
});
```

## PART 6 — Show mode clearly

Add indicator in Selected Job card or banner:

```html
<div id="dry-run-indicator">🧪 Dry Run Mode</div>
```

Update:

```js
function updateDryRunUI(enabled) {
  const el = document.getElementById("dry-run-indicator");
  el.textContent = enabled ? "🧪 Dry Run Mode" : "🚀 Live Mode";
}
```

## PART 7 — Persist state (important)

Use localStorage:

```js
localStorage.setItem("dryRun", enabled);
```

* restore toggle state

## PART 8 — Behavior rules

* Dry run ON:

  * no actual booking clicks
  * full navigation + validation still runs

* Dry run OFF:

  * real booking happens

* Toggle takes effect immediately (no refresh)
