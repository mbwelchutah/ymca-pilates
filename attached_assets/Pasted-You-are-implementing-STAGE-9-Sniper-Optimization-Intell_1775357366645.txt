STAGE 9.1 — SNIPER CONFIDENCE SCORE
================================

Goal:
Provide a single clear confidence signal for booking readiness.

Tasks:
1. Implement a sniper confidence score (0–100%)
2. Derive the score from:
   - session readiness
   - discovery success (recent preflight results)
   - action availability (Reserve vs Waitlist vs blocked)
   - recent failure history
   - timing alignment with booking window
3. Weight recent preflight results more heavily than older data
4. Reduce score if:
   - session unstable
   - discovery failed recently
   - action blocked or unknown
   - repeated failures of same type
5. Output:
   - numeric score (e.g. 92%)
   - short explanation:
     “Session ready, class found, action reachable”

6. Display on Now as:
   - small, calm confidence indicator
   - do not dominate screen

STAGE 9.2 — AUTO PREFLIGHT SYSTEM
================================

Goal:
Automatically validate readiness before booking window.

Tasks:
1. Implement scheduled preflight runs:
   - configurable timing (e.g. 30 min, 10 min, 2 min before window)
2. Preflight should:
   - validate session
   - validate discovery
   - validate action path
3. Update readiness + confidence score after each run
4. Store results in Tools
5. Do not trigger real booking actions
6. Avoid excessive runs (no tight loops)

Optional:
- allow user to enable/disable auto preflight
