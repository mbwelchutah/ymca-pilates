CONTEXT / PROBLEMS TO FIX
==================================================

Known issues from earlier audits:

1. schedule-cache freshness is currently too coarse
- freshness is derived mainly from file-level savedAt / merge time
- this can make old entries appear fresher than they really are

2. classifier freshness may not reflect the specific matched entry age
- a fresh cache merge can still contain stale class rows / stale dates / outdated slot states

3. stale or aging class truth can still influence behavior too strongly
- suppressing warmup
- making Now too confident
- making Plan show stronger preview than evidence supports

We want to make freshness specific to the matched class truth, not just the cache container.
