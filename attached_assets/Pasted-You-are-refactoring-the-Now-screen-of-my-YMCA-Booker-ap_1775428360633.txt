You are refactoring the Now screen of my YMCA Booker app.

PRIMARY GOAL

Make the Now screen feel calm, clear, and trustworthy.
Right now it feels jarring because:
- “Check Now” and “Dry Run” feel duplicative
- too many status cards compete with each other
- the most important message is not visually obvious

This is a mobile-first web app.
The Now screen should behave like a mission-control screen for one active class.

DESIGN DIRECTION

The Now screen should answer, at a glance:
1. What class is targeted?
2. When does registration open?
3. Am I ready?
4. What should I do next?

Keep it clean, iPhone-friendly, and operational.
Do NOT turn it into a dashboard full of technical details.
Do NOT redesign the whole app in one pass.

CORE PRODUCT CHANGE

Replace the confusing “Check Now” + “Dry Run” dual-action model with:

- one primary action:
  - “Run Check”
- one explicit mode control:
  - Test mode
  - Live mode

This creates one action with two modes instead of two competing buttons.

RULES

- Keep the app runnable after every stage
- Do not break existing booking logic
- Do not remove Tools diagnostics
- Do not show raw debug logs on Now
- Stop after each stage and summarize what changed
- Do not jump ahead

STAGE 1 — UNIFY ACTION MODEL
================================

Goal:
Replace the duplicate-feeling “Check Now” and “Dry Run” controls with a clearer model.

Tasks:
1. Remove the separate “Dry Run” button from the main Now action area
2. Keep one primary action button:
   - “Run Check”
3. Add a mode selector above or near the primary action:
   - Test
   - Live
4. Map behavior:
   - Test mode = safe preflight / no booking
   - Live mode = real run / booking-capable
5. If existing Dry Run system already exists, wire the mode selector into it instead of inventing new behavior

Important:
- One action, two modes
- This must reduce confusion, not add more
